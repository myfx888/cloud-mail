# Cloud Mail - 修复 resend_enabled 列缺失错误的实现计划

## 问题分析

系统设置界面提示错误：`D1_ERROR: no such column: resend_enabled: SQLITE_ERROR`

经过代码分析，发现：
1. 在 `mail-worker/src/entity/setting.js` 文件中定义了 `resendEnabled` 列（第57行）
2. 但在数据库迁移代码中没有添加这个字段的语句
3. 导致数据库表结构与代码定义不匹配

## 技术细节

- **数据库类型**: SQLite (D1)
- **迁移方法**: 通过 `ALTER TABLE` 语句添加新字段
- **默认值**: 1（启用状态）
- **字段类型**: INTEGER
- **错误处理**: 迁移方法包含错误捕获，确保即使字段已存在也不会导致迁移失败
