# 邮件签名功能实现计划

## 1. 后端修改

### 1.1 修改账户实体

* 在 `mail-worker/src/entity/account.js` 中添加 `signature` 字段

* 字段类型：text，默认值为空字符串

* 用于存储该账户的默认邮件签名

### 1.2 更新 API 接口

* 修改 `account-api.js` 中的相关接口，支持签名的设置和获取

* 确保签名数据在账户列表和详情中正确返回

### 1.3 更新服务层

* 修改 `account-service.js`，添加签名相关的业务逻辑

* 确保签名数据在账户操作时正确处理

## 2. 前端修改

### 2.1 修改账户设置界面

* 在 `mail-vue/src/views/account/index.vue` 中添加签名设置功能

* 在 SMTP 配置对话框中添加签名设置区域

* 提供文本编辑器用于输入签名内容

### 2.2 修改邮件编辑界面

* 在 `mail-vue/src/layout/write/index.vue` 中添加签名功能

* 自动加载当前发送账户的默认签名

* 提供签名预览和编辑功能

### 2.3 更新 API 调用

* 修改 `setting.js` 中的相关函数，支持签名数据的获取和保存

* 确保签名数据在账户配置中正确传递

## 3. 功能测试

### 3.1 测试签名设置

* 为不同账户设置不同的签名

* 验证签名是否正确保存和加载

### 3.2 测试邮件发送

* 验证邮件发送时是否包含正确的签名

* 测试不同账户发送邮件时的签名效果

### 3.3 测试签名编辑

* 验证签名编辑功能是否正常工作

* 测试签名保存后是否立即生效

## 4. 界面优化

### 4.1 签名编辑器

* 添加基本的文本编辑功能

* 提供签名预览功能

* 支持基本的格式化（如换行、缩进等）

### 4.2 用户体验

* 确保界面布局合理，操作流畅

* 添加适当的提示信息

* 优化签名设置的用户体验

## 实现步骤

1. 首先修改后端账户实体，添加 signature 字段
2. 更新后端 API 接口，支持签名的设置和获取
3. 修改前端账户设置界面，添加签名设置功能
4. 修改邮件编辑界面，添加签名加载和显示功能
5. 测试所有功能是否正常工作
6. 优化界面和用户体验

## 预期效果

* 每个邮件账户可以设置独立的默认签名

* 发送邮件时自动添加该账户的默认签名

* 签名设置界面操作简单直观，用户体验良好
