# 多邮件签名功能任务分解

## 后端任务

### 任务1：修改账户实体，添加多签名支持
- **描述**：修改 `mail-worker/src/entity/account.js`，将单个 `signature` 字段改为 `signatures` 字段，使用 JSON 格式存储多个签名
- **优先级**：P0
- **依赖**：无
- **完成标准**：
  - 账户实体结构正确，支持多签名存储
  - 数据库迁移脚本执行成功

### 任务2：更新数据库初始化文件
- **描述**：更新 `mail-worker/src/init/init.js`，添加数据库迁移逻辑，确保现有数据的平滑迁移
- **优先级**：P0
- **依赖**：任务1
- **完成标准**：
  - init文件更新完成，数据库初始化正常
  - 现有签名数据正确迁移到新结构

### 任务3：实现签名管理 API 接口
- **描述**：在 `account-api.js` 中实现签名的添加、编辑、删除、获取接口
- **优先级**：P0
- **依赖**：任务2
- **完成标准**：
  - 所有签名管理接口返回正确的状态码
  - 接口能够正确处理签名的增删改查操作

### 任务4：更新服务层逻辑
- **描述**：更新 `account-service.js` 和 `setting-service.js`，添加多签名相关的业务逻辑
- **优先级**：P0
- **依赖**：任务3
- **完成标准**：
  - 服务层能够正确处理多签名的业务逻辑
  - 签名数据能够正确保存和加载
