# mailcow 集成功能规格说明

## 1. 功能需求

### 1.1 核心功能
- **自动创建邮件账户**：当用户在系统中新建邮件账户时，系统需自动通过 mailcow API 创建对应的邮件账户
- **同步 SMTP 配置**：将生成的账户资料（包括用户名、密码、SMTP 服务器地址、端口号等信息）完整添加到该邮件账户的 SMTP 账户信息配置中
- **操作状态反馈**：提供操作成功/失败的状态反馈
- **错误处理机制**：实现完善的错误处理机制

### 1.2 配置管理
- **多服务器支持**：在系统设置中添加 mailcow API 地址和 API 密钥配置，支持配置多台服务器
- **默认服务器选择**：提供默认服务器选择功能
- **配置验证**：实现配置验证和测试功能

## 2. 技术实现

### 2.1 技术栈
- **运行环境**：Cloudflare Worker
- **API 调用**：使用 CF Worker 内置的 `fetch` API
- **数据存储**：现有数据库系统
- **错误处理**：实现重试机制和详细的错误信息

### 2.2 模块设计

#### 2.2.4 系统设置扩展

  - 配置验证功能

## 3. 数据流程

### 3.1 账户创建流程
1. 用户提交新建邮件账户请求
2. 系统验证账户信息
3. 创建本地邮件账户
4. 调用 mailcow API 创建对应账户
5. 获取 mailcow 账户信息
6. 更新本地账户的 SMTP 配置
7. 返回操作结果

### 3.2 错误处理流程
1. 捕获 API 调用错误
2. 记录错误信息
3. 尝试重试（可配置）
4. 如果重试失败，返回错误信息
5. 确保本地数据一致性

## 4. 配置项

### 4.1 系统设置
- **mailcowEnabled**：mailcow 功能开关
- **mailcowServers**：mailcow 服务器配置列表
  - **name**：服务器名称
  - **apiUrl**：API 地址
  - **apiKey**：API 密钥
  - **isDefault**：是否默认服务器
- **mailcowRetryCount**：API 调用重试次数
- **mailcowTimeout**：API 调用超时时间

### 4.2 环境变量（备用）
- **MAILCOW_API_URL**：默认 mailcow API 地址
- **MAILCOW_API_KEY**：默认 mailcow API 密钥

## 5. 错误处理

### 5.1 错误类型
- **API 调用错误**：网络问题、服务器错误等
- **认证错误**：API 密钥无效
- **参数错误**：请求参数错误
- **账户创建错误**：邮箱已存在等

### 5.2 错误处理策略
- **重试机制**：网络错误时自动重试
- **详细错误信息**：返回具体的错误原因
- **事务回滚**：确保数据一致性

## 6. 安全考虑

### 6.1 API 密钥安全
- 存储 API 密钥时进行加密

### 6.2 数据安全
