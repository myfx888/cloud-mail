# 邮件导入导出认证问题修复 - 产品需求文档

## 概述
- **摘要**：修复邮件导入导出功能中的身份认证失效问题，并添加批量导入和批量导出功能。
- **目的**：解决用户在使用邮件导入导出功能时遇到的"身份认证失效,请重新登录"错误，同时提升用户体验，支持批量操作。
- **目标用户**：所有使用邮件系统的用户。

## 目标
- 修复邮件导入功能的身份认证问题
- 修复邮件导出功能的身份认证问题
- 添加批量邮件导入功能
- 添加批量邮件导出功能

## 非目标（范围外）
- 不修改现有的认证机制核心逻辑
- 不改变现有的邮件存储结构
- 不涉及其他功能模块的修改

## 背景与上下文
- 现有的邮件导入导出功能使用 `window.open` 和 `fetch` 发送请求，但没有包含认证 token
- 认证机制依赖于请求头中的 `Authorization` 字段，包含 JWT token
- 前端使用 Vue.js 框架，后端使用 Hono 框架在 Cloudflare Worker 上运行

## 功能需求
- **FR-1**：修复邮件导出功能的认证问题
- **FR-2**：修复邮件导入功能的认证问题
- **FR-3**：添加批量邮件导出功能
- **FR-4**：添加批量邮件导入功能

## 非功能需求
- **NFR-1**：认证修复应保持与现有认证机制的一致性
- **NFR-2**：批量操作应保持良好的用户体验，包括进度显示
- **NFR-3**：修改应符合 Cloudflare Worker 的运行环境要求

## 约束
- **技术**：Cloudflare Worker 运行环境，Vue.js 前端框架
- **依赖**：现有的认证机制和 JWT token 管理

## 假设
- 前端已正确存储和管理认证 token
- 后端认证中间件正常工作
- 用户已登录并持有有效的认证 token

## 验收标准

### AC-1：邮件导出功能认证修复
- **给定**：用户已登录系统，持有有效认证 token
- **当**：用户点击导出邮件按钮
- **则**：系统成功导出邮件，不会提示认证失效
- **验证**：`programmatic`

### AC-2：邮件导入功能认证修复
- **给定**：用户已登录系统，持有有效认证 token
- **当**：用户上传 .eml 文件导入邮件
- **则**：系统成功导入邮件，不会提示认证失效
- **验证**：`programmatic`

### AC-3：批量邮件导出功能
- **给定**：用户已登录系统，选择了多个邮件
- **当**：用户点击批量导出按钮
- **则**：系统成功导出所有选中的邮件
- **验证**：`programmatic`

### AC-4：批量邮件导入功能
- **给定**：用户已登录系统，选择了多个 .eml 文件
- **当**：用户点击批量导入按钮
- **则**：系统成功导入所有选中的邮件，并显示导入进度
- **验证**：`programmatic`
