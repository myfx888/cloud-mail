## 新增需求

### 需求:账户界面 SMTP 操作入口统一

系统必须在邮件账户界面中将重复的 SMTP 功能按钮合并为单一入口，避免重复操作路径。

#### 场景:仅显示一个 SMTP 操作按钮
- **当** 用户查看账户列表行操作区
- **那么** 系统必须仅显示一个“SMTP 设置”按钮，禁止出现重复 SMTP 功能按钮

#### 场景:统一入口覆盖原有能力
- **当** 用户点击“SMTP 设置”按钮
- **那么** 系统必须在统一弹窗中提供服务器选择、参数查看和连接测试能力

### 需求:统一入口支持失败状态协同

系统必须确保 SMTP 统一入口与 Mailcow 失败重试入口在同一操作区且状态清晰。

#### 场景:失败账户显示重试与 SMTP 设置
- **当** 账户 `mailcowStatus=failed`
- **那么** 系统必须同时显示“重试”和“SMTP 设置”入口，且按钮语义不重复

#### 场景:进行中状态禁用重复触发
- **当** 用户已触发 SMTP 保存或 Mailcow 重试
- **那么** 系统必须显示 loading 并禁止重复点击

### 需求:统一入口交互一致性

系统必须保证账户列表与账户详情中的 SMTP 操作入口命名和行为一致。

#### 场景:列表和详情命名一致
- **当** 用户在列表页与详情页查看 SMTP 操作入口
- **那么** 入口名称必须一致为“SMTP 设置”

#### 场景:保存后状态一致刷新
- **当** 用户在任一入口保存 SMTP 配置成功
- **那么** 列表和详情必须刷新并显示一致配置

### 需求:回复编辑窗口支持 SMTP 下拉切换

系统必须在邮件回复编辑窗口右下角提供 SMTP 服务器下拉框，支持在多个 SMTP 服务器之间切换。

#### 场景:右下角显示 SMTP 下拉框
- **当** 用户打开邮件回复编辑窗口且存在多个可用 SMTP 服务器
- **那么** 窗口右下角必须显示 SMTP 下拉选择组件

#### 场景:切换 SMTP 服务器
- **当** 用户在右下角下拉框中选择不同 SMTP 服务器
- **那么** 系统必须将当前回复发送配置切换到所选服务器

#### 场景:无可用服务器时禁用切换
- **当** 当前账户无可用 SMTP 服务器可选
- **那么** 下拉框必须禁用并提示用户先完成 SMTP 配置
