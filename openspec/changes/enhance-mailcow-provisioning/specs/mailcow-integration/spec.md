## 修改需求

### 需求:Mailcow 账户自动创建

系统必须在用户添加新邮箱时，自动在 Mailcow 中创建同名账户，并根据配置使用统一密码或随机密码。

#### 场景:启用 Mailcow 且固定密码模式
- **当** 用户添加新邮箱，`mailcowEnabled` 为 `1`，`mailcowPasswordMode` 为 `fixed`
- **那么** 系统调用 Mailcow API 创建账户，使用 `mailcowProvisionPassword` 作为密码

#### 场景:启用 Mailcow 且随机密码模式
- **当** 用户添加新邮箱，`mailcowEnabled` 为 `1`，`mailcowPasswordMode` 为 `random`
- **那么** 系统调用 Mailcow API 创建账户，使用随机生成的密码

#### 场景:禁用 Mailcow
- **当** 用户添加新邮箱，`mailcowEnabled` 为 `0`
- **那么** 系统不调用 Mailcow API，仅创建本地账户

### 需求:Mailcow 多服务器路由

系统必须支持配置多个 Mailcow 服务器，并在账户创建或重试时按账号绑定优先、默认服务器回退的策略选择目标服务器。

#### 场景:账户绑定服务器优先
- **当** 账户存在 `mailcowServerId` 且该服务器可用
- **那么** 系统必须使用该 `mailcowServerId` 对应的 Mailcow 服务器执行 API 操作

#### 场景:未绑定时使用默认服务器
- **当** 账户未配置 `mailcowServerId`
- **那么** 系统必须使用 `mailcowServers` 中 `isDefault=true` 的服务器执行 API 操作

#### 场景:默认服务器唯一性
- **当** 管理员保存 Mailcow 服务器列表
- **那么** 系统必须保证最多仅有一个默认服务器

#### 场景:目标服务器不可用
- **当** 账户绑定服务器不可用或不存在
- **那么** 系统必须返回失败并记录原因，禁止自动切换到其他非默认服务器

### 需求:SMTP 配置自动回填

系统必须在 Mailcow 账户创建成功（或已存在）后，自动将 SMTP 配置回填到本地账户。

#### 场景:SMTP 回填使用服务器配置
- **当** Mailcow 账户创建成功，服务器配置包含 SMTP 字段
- **那么** 系统使用服务器配置更新本地账户的 SMTP 设置

#### 场景:SMTP 回填使用全局模板
- **当** Mailcow 账户创建成功，服务器配置缺少 SMTP 字段
- **那么** 系统使用全局 SMTP 模板补充缺失字段，更新本地账户的 SMTP 设置

#### 场景:SMTP 回填使用默认值
- **当** Mailcow 账户创建成功，服务器配置和全局模板均缺少某字段
- **那么** 系统使用硬编码默认值补充，更新本地账户的 SMTP 设置

### 需求:敏感字段脱敏返回

系统必须在查询系统设置时，对敏感字段脱敏返回。

#### 场景:API Key 脱敏
- **当** 查询系统设置返回 `mailcowServers`
- **那么** 每个服务器的 `apiKey` 字段显示为 `key****xxx`（前4后3字符）

#### 场景:统一密码脱敏
- **当** 查询系统设置返回 `mailcowProvisionPassword`
- **那么** 字段值显示为 `******`
