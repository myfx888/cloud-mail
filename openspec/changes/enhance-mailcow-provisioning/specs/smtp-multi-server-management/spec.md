## 新增需求

### 需求:管理员可配置 SMTP 多服务器

系统必须允许管理员在系统设置中维护多个 SMTP 服务器配置，并支持新增、编辑、禁用和设置默认服务器。

#### 场景:新增 SMTP 服务器
- **当** 管理员在设置中新增 SMTP 服务器并保存
- **那么** 系统必须将该服务器写入 `smtpServers` 配置列表

#### 场景:设置默认 SMTP 服务器
- **当** 管理员将某服务器设置为默认
- **那么** 系统必须保证仅有一个服务器 `isDefault=true`

#### 场景:禁用 SMTP 服务器
- **当** 管理员禁用某 SMTP 服务器
- **那么** 系统必须禁止新账户自动绑定该服务器

### 需求:账户支持绑定 SMTP 服务器

系统必须允许账户绑定指定 SMTP 服务器，并在绑定后自动同步 SMTP 参数。

#### 场景:新账户绑定默认服务器
- **当** 新增账户且未显式选择服务器
- **那么** 系统必须自动绑定默认 SMTP 服务器并写入 SMTP 参数

#### 场景:账户切换 SMTP 服务器
- **当** 管理员为账户切换 `smtpServerId`
- **那么** 系统必须立即应用目标服务器配置并更新账户 SMTP 字段

#### 场景:绑定服务器不可用
- **当** 账户绑定的 `smtpServerId` 对应服务器被删除或禁用
- **那么** 系统必须提示配置异常并要求重新选择可用服务器

### 需求:SMTP 服务器字段校验

系统必须对 SMTP 服务器配置进行字段校验，禁止保存无效配置。

#### 场景:必填字段校验
- **当** 管理员保存 SMTP 服务器配置缺少 `name` 或 `smtpHost`
- **那么** 系统必须拒绝保存并返回错误提示

#### 场景:端口与加密方式校验
- **当** 管理员保存 `smtpPort` 非法或 `smtpSecure` 值超出定义范围
- **那么** 系统必须拒绝保存并返回错误提示

### 需求:管理员可在配置中心统一管理服务器

系统必须提供配置中心化界面，以服务器为核心组织 Mailcow 与 SMTP 相关配置，并支持按服务器粒度测试与风险拦截。

#### 场景:按服务器测试连接
- **当** 管理员在配置中心对某个服务器执行测试连接
- **那么** 系统必须返回该服务器最新测试结果（成功/失败、时间、摘要）并在列表中可见

#### 场景:禁用默认服务器拦截
- **当** 管理员尝试禁用当前默认服务器且未指定新的默认服务器
- **那么** 系统必须阻止操作并提示先设置新的默认服务器

#### 场景:删除有依赖服务器拦截
- **当** 管理员尝试删除仍被账户绑定的服务器
- **那么** 系统必须阻止删除并展示受影响账户数量

#### 场景:展示服务器影响范围
- **当** 管理员查看服务器详情
- **那么** 系统必须显示该服务器的绑定账户数量以辅助风险评估
