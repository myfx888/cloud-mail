## 新增需求

### 需求:重复账户检测

系统必须在创建 Mailcow 账户前检测邮箱是否已存在，避免重复创建。

#### 场景:邮箱不存在
- **当** 调用 Mailcow API 查询邮箱返回不存在
- **那么** 系统继续执行创建流程

#### 场景:邮箱已存在
- **当** 调用 Mailcow API 查询邮箱返回已存在
- **那么** 系统跳过创建步骤，直接执行 SMTP 回填流程

### 需求:已存在邮箱视为成功

系统必须将已存在的 Mailcow 邮箱视为成功状态，不报错。

#### 场景:已存在邮箱状态标记
- **当** 检测到邮箱已存在
- **那么** 账户的 `mailcowStatus` 字段标记为 `exists_as_success`

#### 场景:已存在邮箱 SMTP 回填
- **当** 检测到邮箱已存在
- **那么** 系统继续执行 SMTP 配置回填，使用服务器配置或全局模板

### 需求:重复账户检测失败处理

系统必须处理 Mailcow API 查询失败的情况，不阻断主流程。

#### 场景:查询 API 失败
- **当** 调用 Mailcow API 查询邮箱失败（网络错误、超时等）
- **那么** 系统记录警告日志，继续执行创建流程（假设邮箱不存在）

#### 场景:查询 API 返回异常格式
- **当** Mailcow API 返回格式不符合预期
- **那么** 系统记录警告日志，继续执行创建流程
