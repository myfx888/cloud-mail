## 新增需求

### 需求:管理员可配置创建失败策略

系统必须允许管理员在系统设置中配置 Mailcow 账户创建失败时的处理策略，支持宽松模式（`0`）和严格模式（`1`）。

#### 场景:配置宽松模式
- **当** 管理员设置 `mailcowCreateStrict` 为 `0`
- **那么** Mailcow 创建失败时，本地账户仍创建成功，仅记录失败状态

#### 场景:配置严格模式
- **当** 管理员设置 `mailcowCreateStrict` 为 `1`
- **那么** Mailcow 创建失败时，本地账户创建也失败，返回错误信息

### 需求:宽松模式下记录失败状态

系统必须在宽松模式下记录 Mailcow 创建失败的状态，便于后续排查。

#### 场景:宽松模式失败记录
- **当** Mailcow 创建失败且 `mailcowCreateStrict` 为 `0`
- **那么** 账户的 `mailcowStatus` 字段记录为 `failed`，并记录失败原因

### 需求:严格模式下阻断本地创建

系统必须在严格模式下阻断本地账户创建，当 Mailcow 创建失败时。

#### 场景:严格模式失败阻断
- **当** Mailcow 创建失败且 `mailcowCreateStrict` 为 `1`
- **那么** 本地账户创建失败，返回错误信息，不保存账户记录

#### 场景:严格模式成功创建
- **当** Mailcow 创建成功且 `mailcowCreateStrict` 为 `1`
- **那么** 本地账户创建成功，并回填 SMTP 配置

### 需求:默认宽松模式

系统必须将宽松模式作为默认策略，保证兼容性。

#### 场景:未配置时默认宽松
- **当** `mailcowCreateStrict` 字段未配置或为空
- **那么** 系统默认使用宽松模式（值 `0`）

### 需求:失败项支持手动重试

系统必须为 Mailcow 配置失败项提供重试按钮，允许管理员按账号触发重试。

#### 场景:失败项显示重试按钮
- **当** 账户的 `mailcowStatus` 为 `failed`
- **那么** 界面显示“重试”按钮并允许管理员触发重试

#### 场景:非失败项不显示重试按钮
- **当** 账户的 `mailcowStatus` 不为 `failed`
- **那么** 界面禁止显示“重试”按钮

#### 场景:重试成功
- **当** 管理员点击“重试”且 Mailcow 重试成功
- **那么** 系统将 `mailcowStatus` 更新为 `success` 或 `exists_as_success`，并回填 SMTP 配置

#### 场景:重试失败
- **当** 管理员点击“重试”且 Mailcow 重试失败
- **那么** 系统保持 `mailcowStatus` 为 `failed`，并更新失败原因
