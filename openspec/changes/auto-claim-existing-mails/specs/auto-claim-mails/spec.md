## 新增需求

### 需求:自动认领匹配新邮箱地址的历史邮件
系统必须在新邮箱账号创建成功后，自动查找并关联那些在账号创建之前已经到达系统、且由于收件人不存在而被标记为“无人收件”（NOONE）的历史邮件。

#### 场景:成功认领历史邮件
- **当** 用户创建一个新的邮箱地址 `test@example.com`
- **那么** 系统必须查找 `email` 表中 `toEmail` 为 `test@example.com`（忽略大小写）、`status` 为 `NOONE`、`userId` 为 `0` 且 `accountId` 为 `0` 的所有记录
- **那么** 系统必须将这些记录的 `userId` 更新为新账号的 `userId`，`accountId` 更新为新账号的 `accountId`，并将 `status` 更新为 `RECEIVE` (0)
- **那么** 系统必须同步更新 `att` 表中对应这些邮件的所有附件记录，将其 `userId` 和 `accountId` 也更新为新账号的信息

#### 场景:无匹配邮件时不执行认领
- **当** 用户创建一个新的邮箱地址 `new@example.com`，且系统中不存在发往该地址的 `NOONE` 邮件时
- **那么** 系统必须正常完成账号创建逻辑，且认领步骤不应对任何现有数据产生副作用
