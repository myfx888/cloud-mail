## 修改需求

### 需求:SMTP 验证流程必须与发信流程使用一致的安全模式
SMTP 验证接口必须复用与发信流程一致的安全模式解析规则，禁止验证成功而发信失败（或反之）的模式偏差。

#### 场景:验证 `smtpSecure=2`
- **当** 验证参数中 `smtpSecure` 为 `2`
- **那么** 验证连接必须使用 `secure=false` 且 `startTls=true`

### 需求:SMTP 验证连接必须可回收
SMTP 验证流程必须在请求结束时关闭已建立连接，禁止留下悬挂连接。

#### 场景:验证成功后连接关闭
- **当** SMTP 连接验证成功
- **那么** 系统必须在返回结果前或返回后立即关闭该连接
