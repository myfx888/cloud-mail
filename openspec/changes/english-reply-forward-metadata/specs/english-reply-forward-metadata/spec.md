## 新增需求

### 需求:回复元信息固定英文
系统在生成回复邮件引用头时，必须固定输出英文元信息，禁止随当前界面语言切换。

#### 场景:中文界面下发起回复
- **当** 用户界面语言为中文并执行“回复”操作
- **那么** 引用头中的时间必须采用英文格式，且动词必须为 `wrote:`

#### 场景:英文界面下发起回复
- **当** 用户界面语言为英文并执行“回复”操作
- **那么** 引用头中的时间与动词必须保持英文格式，且与中文界面下生成结果一致（除原始正文外）

### 需求:转发元信息使用标准英文头
系统在生成转发内容时，必须包含标准英文转发头，且头字段不受当前界面语言影响。

#### 场景:生成转发模板
- **当** 用户执行“转发”操作
- **那么** 模板中必须包含英文分隔头与 `From`、`Date`、`Subject`、`To` 字段

#### 场景:中文界面下转发
- **当** 用户界面语言为中文并执行“转发”操作
- **那么** 转发头字段名必须仍为英文，且日期必须为英文格式

### 需求:回复与转发主题前缀统一英文
系统必须对回复与转发主题前缀执行去重与标准化，最终输出英文前缀。

#### 场景:回复主题缺少前缀
- **当** 原主题不包含任何回复前缀
- **那么** 新主题必须以 `Re: ` 开头

#### 场景:转发主题缺少前缀
- **当** 原主题不包含任何转发前缀
- **那么** 新主题必须以 `Fwd: ` 开头

#### 场景:原主题包含中文或英文等价前缀
- **当** 原主题已包含 `Re:`、`回复:`、`Fwd:`、`Fw:` 或 `转发:` 等等价前缀
- **那么** 系统必须避免重复叠加前缀，并输出单一英文前缀语义

### 需求:正文内容保持原始语言与内容
该改造必须仅作用于元信息，禁止修改被引用或被转发的原始正文内容。

#### 场景:原邮件正文为中文
- **当** 用户回复或转发一封中文正文邮件
- **那么** 生成内容中的原正文必须保持中文原文，不得被翻译、替换或重写

#### 场景:原邮件正文包含 HTML 与纯文本
- **当** 原邮件存在 HTML 内容或纯文本回退内容
- **那么** 系统必须沿用既有正文渲染路径，仅替换元信息头部文本
