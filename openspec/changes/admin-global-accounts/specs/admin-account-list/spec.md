## 新增需求

### 需求:管理员全局邮箱列表 API
系统必须提供 `GET /admin/accounts` 端点，允许管理员分页查询所有用户的邮箱账户。该端点必须仅对拥有 `user:query` 权限的已认证用户开放。

#### 场景:成功获取全局邮箱列表
- **当** 管理员请求 `GET /admin/accounts?num=1&size=15`
- **那么** 系统返回包含 `list` 和 `total` 的响应，`list` 中每条记录必须包含 `accountId`、`email`、`userId`、`userEmail`、`status`、`createTime`、`smtpOverride`、`mailcowServerId`、`smtpAccountCount` 字段

#### 场景:按邮箱地址搜索
- **当** 管理员请求 `GET /admin/accounts?email=alice&num=1&size=15`
- **那么** 系统必须返回邮箱地址中包含 "alice" 的账户（模糊匹配，大小写不敏感）

#### 场景:按 SMTP 状态过滤 - 已配置
- **当** 管理员请求 `GET /admin/accounts?smtpStatus=configured&num=1&size=15`
- **那么** 系统必须仅返回 `smtpAccountCount > 0` 的邮箱账户

#### 场景:按 SMTP 状态过滤 - 未配置
- **当** 管理员请求 `GET /admin/accounts?smtpStatus=unconfigured&num=1&size=15`
- **那么** 系统必须仅返回 `smtpAccountCount = 0` 的邮箱账户

#### 场景:非管理员访问被拒绝
- **当** 不具有 `user:query` 权限的用户请求 `GET /admin/accounts`
- **那么** 系统必须返回 403 错误

#### 场景:分页参数限制
- **当** 请求的 `size` 超过 30
- **那么** 系统必须将 `size` 限制为 30

### 需求:管理员全局邮箱列表前端展示
系统必须在管理员邮件账户页面 (`account/index.vue`) 的现有邮箱列表下方展示「所有用户邮箱」区域。该区域必须仅对管理员用户可见。

#### 场景:管理员查看全局邮箱列表
- **当** 管理员打开邮件账户页面
- **那么** 页面在「我的邮件账户」卡片下方必须显示「所有用户邮箱」卡片，其中包含分页表格，表格列为：邮箱、所属用户、SMTP 状态、操作

#### 场景:普通用户不可见
- **当** 非管理员用户打开邮件账户页面
- **那么** 页面必须不显示「所有用户邮箱」区域

### 需求:SMTP 状态展示
全局邮箱列表中的每一行必须展示该邮箱的 SMTP 配置状态。

#### 场景:显示 Mailcow 已配置状态
- **当** 邮箱的 `smtpAccountCount > 0` 且 `mailcowServerId` 不为空
- **那么** 必须显示「已配置 (Mailcow)」标签

#### 场景:显示手动已配置状态
- **当** 邮箱的 `smtpAccountCount > 0` 且 `mailcowServerId` 为空
- **那么** 必须显示「已配置 (手动)」标签

#### 场景:显示未配置状态
- **当** 邮箱的 `smtpAccountCount = 0`
- **那么** 必须显示「未配置」标签

### 需求:全局邮箱搜索与过滤
管理员必须能够在全局邮箱列表中按邮箱地址搜索和按 SMTP 状态过滤。

#### 场景:搜索邮箱
- **当** 管理员在搜索框中输入关键词并触发搜索
- **那么** 表格必须仅显示邮箱地址匹配该关键词的结果，分页重置到第一页

#### 场景:按 SMTP 状态过滤
- **当** 管理员从下拉选择器中选择「已配置」或「未配置」
- **那么** 表格必须仅显示符合所选 SMTP 状态的结果

### 需求:全局邮箱操作 - SMTP 设置
管理员必须能够从全局邮箱列表中直接打开任意邮箱的 SMTP 账户管理器。

#### 场景:打开 SMTP 设置
- **当** 管理员点击某行的「SMTP 设置」操作
- **那么** 系统必须打开 `smtp-account-manager` 组件弹窗，传入该邮箱的 `accountId`

### 需求:全局邮箱操作 - 一键开通
管理员必须能够从全局邮箱列表中为单个邮箱一键开通 Mailcow SMTP。

#### 场景:一键开通成功
- **当** 管理员点击某行的「一键开通」操作
- **那么** 系统必须调用 `/smtp/provision-mailcow` API 为该邮箱开通 SMTP，成功后刷新列表并显示成功提示

#### 场景:一键开通入口可见性
- **当** 系统未启用 Mailcow 功能
- **那么** 操作下拉中必须不显示「一键开通」选项

### 需求:全局邮箱操作 - 删除
管理员必须能够从全局邮箱列表中删除用户的邮箱账户。

#### 场景:删除邮箱账户
- **当** 管理员点击某行的「删除」操作
- **那么** 系统必须弹出确认对话框，确认后调用 `/user/deleteAccount` API 删除该邮箱并刷新列表
