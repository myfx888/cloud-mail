## 上下文

系统目前拥有邮件账户（Account）的管理功能，但在权限控制层面存在断层：
1. **静态路由**：`/account` 路由在前端是静态定义的，任何登录用户通过输入 URL 均可尝试访问。
2. **映射缺失**：前端的权限路由生成器（`perm.js`）中没有 `account:query` 的映射，导致该页面无法根据角色权限动态显示在侧边栏菜单中。
3. **权限树不直观**：虽然数据库中有权限定义，但由于缺少翻译和类型明确，在角色管理界面展示不够友好。

## 目标 / 非目标

**目标：**
- 将 `/account` 路由完全接入权限管控体系。
- 实现菜单项“邮件账户”根据用户角色动态显示。
- 确保相关操作按钮（添加、删除）受按钮级权限控制。
- 完善权限树的国际化展示。

**非目标：**
- 不修改现有的发信或邮件查看权限逻辑。
- 不修改后端账户管理的具体业务实现。

## 决策

1. **路由迁移**：
   - 将 `mail-vue/src/router/index.js` 中的 `/account` 配置移出静态路由数组。
   - 在 `mail-vue/src/perm/perm.js` 的 `routers` 对象中添加 `account:query` 键值对，指向账户管理页配置。
   - **理由**：统一通过 `permsToRouter` 逻辑生成侧边栏菜单，实现真正意义上的动态权限。

2. **数据库结构与类型明确**：
   - 使用 `mail-worker/src/init/init.js` 进行版本迁移（`v3_9DB`）。
   - 确保 `perm_id: 21` (邮箱侧栏) 的 `type` 为 0 (MENU)，`account:query` 的 `type` 为 0 (MENU)，而 `account:add`/`account:delete` 的 `type` 为 2 (BUTTON)。
   - **理由**：后端权限树逻辑依赖 `type` 字段区分菜单和操作项。

3. **权限词条国际化**：
   - 在 `mail-worker/src/i18n/` 的 `zh.js` 和 `en.js` 中补齐 `perms` 命名空间下的词条。
   - **理由**：提升管理员配置角色时的易用性。

4. **发件逻辑权限复用**：
   - 写信页已有的 `accountStore.currentAccount` 逻辑保持不变，但需确保账户切换受 `account:query` 权限影响。

## 风险 / 权衡

- **[风险] 既有用户权限失效** → **缓解措施**：在 `v3_9DB` 迁移中，自动为已拥有旧版权限 ID 的角色关联新的菜单权限位。
- **[风险] 侧边栏图标不统一** → **缓解措施**：在 `perm.js` 的路由配置中显式指定图标。
