## 上下文

当前 header 组件 (`layout/header/index.vue`) 的 `.toolbar` 区域包含：主题切换按钮（太阳/月亮图标）、通知按钮、用户头像下拉。组件中已存在 `changeLang()` 函数和 `settingStore.lang` 状态，但缺少触发 UI。i18n 基础设施（vue-i18n、zh.js/en.js、settingStore.lang）已完备。

## 目标 / 非目标

**目标：**
- 在主题切换按钮左侧添加一个语言切换按钮
- 复用已有的 `changeLang()` 函数和 `settingStore.lang`
- 样式与现有 `.icon-item` 一致，视觉协调

**非目标：**
- 不支持两种以上语言（仅中/英）
- 不修改 i18n 架构或翻译文件内容
- 不涉及后端变更

## 决策

1. **按钮形式：文字标签而非图标**
   - 使用 "中" / "EN" 文字显示当前语言，点击切换到另一种
   - 理由：语言切换用文字标识比图标更直观，用户一眼就能知道当前语言
   - 替代方案：使用地球图标 + 下拉菜单 — 对于仅两种语言过于复杂

2. **放置位置：主题切换按钮左侧**
   - 在 `.toolbar` 中，dark/light 图标之前插入
   - 理由：用户要求"风格切换按钮左边"，且该位置不影响现有布局

3. **复用现有逻辑**
   - 直接调用已有的 `changeLang()` 函数
   - 通过 `settingStore.lang` 判断当前语言来决定显示文字
   - 理由：零新增状态/逻辑，最小化改动

## 风险 / 权衡

- **风险**：移动端空间可能较紧 → 缓解：按钮使用与 `.icon-item` 相同的 30×30 尺寸，不额外占用空间
- **权衡**：仅支持两种语言的简单切换，如果未来需要更多语言需要改为下拉菜单 → 当前需求明确只需中英文，保持简单
