## 新增需求

### 需求:语言切换按钮
系统必须在右上角工具栏中提供一个语言切换按钮，位于主题切换按钮（太阳/月亮图标）的左侧。该按钮必须允许用户在中文和英文之间切换界面语言。

#### 场景:按钮显示当前语言标识
- **当** 当前语言为中文时
- **那么** 按钮必须显示 "EN" 文字，表示点击后可切换到英文

#### 场景:按钮显示英文状态标识
- **当** 当前语言为英文时
- **那么** 按钮必须显示 "中" 文字，表示点击后可切换到中文

#### 场景:点击切换语言
- **当** 用户点击语言切换按钮
- **那么** 界面语言必须立即切换到另一种语言，且按钮文字同步更新

### 需求:按钮样式一致性
语言切换按钮的尺寸和交互样式必须与工具栏中其他图标按钮（`.icon-item`）保持一致，包括尺寸（30×30）、圆角、hover 效果。

#### 场景:hover 效果
- **当** 用户将鼠标悬停在语言切换按钮上
- **那么** 按钮必须显示与其他工具栏图标相同的 hover 背景色

### 需求:按钮位置固定
语言切换按钮必须始终渲染在主题切换按钮的左侧，禁止出现在其他位置。

#### 场景:工具栏布局顺序
- **当** 页面加载完成
- **那么** 工具栏从左到右的顺序必须为：语言切换按钮 → 主题切换按钮 → 通知按钮 → 用户头像
