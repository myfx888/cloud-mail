## 新增需求

### 需求:管理员可选展示域名
系统必须允许管理员在后台设置中选择哪些域名后缀在前端（登录页、注册页、添加邮箱页）显示。

#### 场景:选择部分域名显示
- **当** 管理员在设置中勾选了 "@a.com" 和 "@b.com"（共有 a, b, c 三个域名）并保存
- **那么** 退出登录后，登录页面的域名下拉列表中仅显示 "@a.com" 和 "@b.com"

#### 场景:未配置显示域名时默认全部显示
- **当** 管理员未在设置中勾选任何域名，或者配置项为空时
- **那么** 登录页面的域名下拉列表必须显示环境变量 `domain` 中定义的所有域名

### 需求:后端过滤公开配置
后端在返回 `websiteConfig`（公开配置）时，必须根据数据库中的 `login_domains` 字段对 `domainList` 进行实时过滤。

#### 场景:后端过滤逻辑验证
- **当** 环境变量 `domain` 为 `["a.com", "b.com", "c.com"]`，数据库 `login_domains` 为 `"a.com,c.com"`
- **那么** 调用 `websiteConfig` 接口返回的 `domainList` 必须为 `["@a.com", "@c.com"]`

### 需求:设置界面域名来源同步
管理后台的域名选择 UI 必须实时反映当前环境变量 `domain` 中定义的最新域名列表。

#### 场景:环境变量变更后的展示
- **当** 环境变量 `domain` 增加了一个新域名 "d.com"
- **那么** 管理后台的“登录域名显示设置”列表中必须自动出现 "@d.com" 供管理员勾选
